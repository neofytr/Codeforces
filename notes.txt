Let n >= 1
Let a[1, n] be an array of non-negative integers that each fit in 64 bits.

Suppose we want to find the following:
A = max(1 <= i <= j <= n, a[i] ^ ... ^ a[j])

Let p[0] = 0 and p[i] = p[i - 1] ^ a[i] for 1 <= i <= n
Then, A = max(1 <= i <= j <= n, p[j] ^ p[i - 1])

The standard way to find A is using binary trie:

--------------------------------------------------------
// each trie node stores two children:
// nxt[0] -> bit 0
// nxt[1] -> bit 1
struct Node {
	Node *nxt[2];

	Node() {
		nxt[0] = nxt[1] = nullptr;
	}
};

class BinaryTrie {
private:
	Node *root;

public:
	BinaryTrie() {
		root = new Node();
	}

	// inserts a number into the trie
	void insert(int x) {
		Node *cur = root;

		// we go from most significant bit -> least significant bit
		// because larger bits contribute more to the xor value
		for (int b = 62; b >= 0; b--) {

			// extract current bit
			int bit = (x >> b) & 1;

			// create path if it doesn't exist
			if (cur->nxt[bit] == nullptr)
				cur->nxt[bit] = new Node();

			// move downward
			cur = cur->nxt[bit];
		}
	}

	// returns maximum possible value of (x ^ y)
	// where y is some number already inserted
	int max_xor(int x) {
		Node *cur = root;

		int ans = 0;

		// again process bits from msb -> lsb
		for (int b = 62; b >= 0; b--) {

			int bit = (x >> b) & 1;

			// for xor:
			// 0 ^ 1 = 1
			// 1 ^ 0 = 1
			//
			// so ideally we want opposite bit
			int want = bit ^ 1;

			// if opposite bit exists,
			// we can make this xor bit = 1
			if (cur->nxt[want] != nullptr) {

				// set this bit in answer
				ans |= (1LL << b);

				// move to preferred branch
				cur = cur->nxt[want];
			}

			// otherwise we are forced to take same bit
			else {
				cur = cur->nxt[bit];
			}
		}

		return ans;
	}
};

signed main() {
	int n;
	cin >> n;

	vector<int> a(n + 1);

	for (int i = 1; i <= n; i++)
		cin >> a[i];

	BinaryTrie trie;

	// prefix xor
	int px = 0;

	// important:
	// insert p[0] = 0 first
	//
	// because subarrays can start from index 1
	trie.insert(0);

	int ans = 0;

	for (int i = 1; i <= n; i++) {

		// current prefix xor
		px ^= a[i];

		// find best previous prefix xor
		// that maximizes:
		//
		// px ^ previous_prefix
		//
		// which equals some subarray xor
		ans = max(ans, trie.max_xor(px));

		// now insert current prefix xor
		// so future positions can use it
		trie.insert(px);
	}

	cout << ans << '\n';
}

3. 

When doing DP/string problems where transitions depend on substrings ending at position i:

	dp[i] depends on substrings s[j...i]

we usually iterate:

	for (int j = i; j >= 1; j--)

Trying to build the substring manually is inefficient because:

	tmp = s[j] + tmp

requires insertion at the front of the string, which is costly.

A better approach is to store all dictionary/pattern strings in a trie in REVERSED order.

Example:
	"abc" is stored as:
		c -> b -> a

Now while iterating backwards from i to 1:

	for (int j = i; j >= 1; j--)

we can walk the trie directly using s[j].

This naturally matches the reverse traversal order and avoids:
	- building temporary substrings
	- front insertions
	- repeated hashing/comparisons

Pattern:
	- substring ends at i
	- iterate j backwards
	- store dictionary strings reversed in trie
	- traverse trie while moving j backwards

This converts substring existence checks from:
	O(length of substring construction/check)

to:
	O(1) trie transition per character.

Very common in:
	- word break DP
	- dictionary DP
	- suffix-based transitions
	- substring existence queries
	- automaton/string DP problems

THE DSU LINE-COMPRESSION TRICK (Monotonic Range/Set Deletions via Path Compression)

1. ABSTRACT MATHEMATICAL PROBLEM

Given a bounded, totally ordered universe U = {0, 1, ..., n, n+1}.
We maintain a dynamic subset S ⊆ U under a sequence of permanent deletions (S ← S \ {x}).
We want to support directional proximity queries on S in O(α(n)) amortized time:
    - Predecessor:  pred_S(x) = max { y ∈ S | y ≤ x }
    - Successor:    succ_S(x) = min { y ∈ S | y ≥ x }

2. THE DSU COCHLEATE REDUCTION

Instead of an undirected equivalence relation, we treat the DSU tree as a 
*directed functional graph* tracking active boundaries. 

Let parent[x] represent the immediate directional proxy for x.
By utilizing ONLY path compression (no Union-by-Rank/Size), the directional 
edges preserve the natural total order of U, creating flat, star-like structures 
pointing directly to the closest active representative.

3. UNIFIED IMPLEMENTATION MATRIX

  Feature               | Predecessor Variant (←)      | Successor Variant (→)
  ----------------------|------------------------------|------------------------
  Target Query          | pred(x) = max(y ∈ S : y ≤ x)  | succ(x) = min(y ∈ S : y ≥ x)
  Sentinel/Null Value   | 0                            | n + 1
  Initial Active Domain | S = {1, 2, ..., n}           | S = {1, 2, ..., n}
  Initialization        | parent[x] = x                | parent[x] = x
  Core Invariant        | find(x) == pred_S(x)         | find(x) == succ_S(x)
  Deletion (Delete x)   | parent[x] = find(x - 1)      | parent[x] = find(x + 1)

4. FORMAL MECHANICS

● Initialization:
  ∀ x ∈ {1...n}, parent[x] = x. (Every element is its own representative/alive).

● The Query / Path Compression Engine:
  The standard DSU `find` function recursively discovers the current representative 
  of x's component and flattens the pointer path dynamically:

  def find(x):
      if parent[x] == x:
          return x
      parent[x] = find(parent[x])  # Path compression memoization
      return parent[x]

● Deletion (The Routing Step):
  When x is removed from S, it can no longer represent itself. Its directional
  pointer is permanently bound to the representative of its immediate neighbor.
  
  For Predecessor: parent[x] = find(x - 1)
  For Successor:   parent[x] = find(x + 1)

  Intuition: Deletion creates a "sinkhole" at x. Any subsequent query landing on 
  or traversing x is instantly warped to the next available valid element.

5. VISUAL MECHANICS (Successor Example for U = {1, 2, 3, 4, 5})

Initial:     [1]   [2]   [3]   [4]   [5]     (all parent[x] = x)
Delete 3:    [1]   [2]   [3]→[4]   [5]       (parent[3] = find(4) = 4)
Delete 4:    [1]   [2]   [3]→[4]→[5]         (parent[4] = find(5) = 5)
Query succ(3): 
  - find(3) visits 3 -> 4 -> 5.
  - Path compression kicks in: parent[3] = 5, parent[4] = 5.
  Graph updates to:
             [3] ──┐
                   ▼
             [4] ─> [5]   [1]   [2]

6. CORRECTNESS & AMORTIZATION INSIGHTS

● Invariant Induction:
  - Base Case: Initially, x ∈ S ⇒ find(x) = x = succ_S(x). Correct.
  - Inductive Step: Assume invariant holds. When x is deleted, succ_S(x) becomes
    succ_S(x + 1) = find(x + 1). Setting parent[x] = find(x + 1) preserves 
    find(x) == succ_S(x) for all future lookups.

● Why Union-by-Rank is Omitted:
  Standard DSU requires Union-by-Rank to avoid deep trees O(N). Here, trees are 
  strictly directional along a monotonic sequence. Because we only link `x` to 
  `find(x ± 1)`, the maximum path length without compression is equal to the length 
  of consecutive deleted segments. Path compression alone guarantees an amortized 
  cost of O(α(n)) per operation, as edges never cross or form structural cycles.

7. PATTERN RECOGNITION & APPLICATION TEMPLATES

Deploy this trick when a problem satisfies the following four-point footprint:
1. Contiguous Domain: Elements map cleanly to an integer index array.
2. Monotonic State: Elements are only removed/consumed, never reinserted.
3. Next-Available Semantics: "Find the first unvisited index ≥ i", "Allocate the closest free slot ≤ weight".
4. Offline / Right-to-Left processing: Can convert complex range updates into sequential point deletions.

Common LeetCode / Codeforces Paradigms:
- Interval Painting: Paint segments [L, R] with color C. Process queries in reverse order 
  (latest color wins). Use successor DSU to skip already-painted indices completely.
- Activity Scheduling / Greedy Deadline Allocation: Given jobs with deadlines, try to place 
  each job at its deadline. If taken, place at predecessor DSU `find(deadline)`.
- Sieve Optimization: Quickly advance to the next un-eliminated element during skips.

7. Counting Sort

This is a non-comparison-based sorting algorithm for sorting distinct keys from a bounded integer domain.

Given:

i. input array A of length n
ii. keys in the range [0, k]

counting sort constructs the sorted order using frequency and prefix sums instead of comparisons.

The idea is:

For each key x:
i. count occurrences of x in A
ii. determine the interval occupied by x in the sorted order
iii. place elements directly into their final positions.

Define:

c[i] = number of elements equal to i in A, for 0 <= i <= K

To construct it, we use:

vector<int> c(k + 1, 0);
for (int i = 1; i <= n; i++)
	c[A[i]]++;

We then transform this array c using prefix sums as:
c[i] <- c[i] + c[i - 1] for 1 <= i <= k

for (int i = 1; i <= k; i++) c[i] += c[i - 1];

After transformation, c[i] = number of elements in A <= i for 0 <= i <= k

Thus, c[i] = position of key i in the final sorted array

To construct the sorted array A'[1, n] stably, we do the following:

for (int i = n; i >= 1; i--)
	A'[c[A[i]]--] = A[i];

The backward traversal ensures stabilty in the sorting procedure.

It may be such that the elements of A need to be sorted according to the value of some function f on the elements. In this case, these values become the key and must be bounded. To stably sort the array in this case, just replace A[i] in the above algorithm by f(A[i]).

9.

Let s[1, n] is a string for n >= 1
Let sa[1, n] is the suffix array of the string s
Let lcp[1, n] is the lcp array of the string s

The number of different substrings of the string s, denoted sub(s), is given by:

sub(s) = (n * (n + 1)) / 2 - summation (lcp[i] for 1 <= i <= n)

Proof:

Every substring of s is a prefix of some suffix of s.
Let the suffixes in lexicographic order be

    s[sa[1]..], s[sa[2]..], ..., s[sa[n]..]

Consider the suffix starting at j = sa[i].
This suffix has n - j + 1 prefixes, so naively it contributes that many substrings.

Now we remove the prefixes that were already counted from previous suffixes.

Among all previous suffixes, the one with the largest common prefix with s[sa[i]..] is exactly s[sa[i-1]..].

Therefore, the number of prefixes already counted is exactly lcp[i]

where lcp[i] = LCP(s[sa[i]..], s[sa[i-1]..]).

Why is considering only the previous suffix enough?

Suppose there were an earlier suffix s[sa[o]..] with o < i - 1 sharing a longer common prefix with s[sa[i]..] than s[sa[i-1]..].

Since suffixes are sorted lexicographically,

    s[sa[o]..] < s[sa[i-1]..] < s[sa[i]..]

If s[sa[o]..] and s[sa[i]..] share a prefix of length k,
then every suffix between them in sorted order must also share that same prefix of length k. In particular, s[sa[i-1]..] would too.

This contradicts the definition of lcp[i].

Hence, the duplicated prefixes are exactly the first lcp[i] prefixes, so the number of new distinct substrings contributed by this suffix is

    (n - sa[i] + 1) - lcp[i]

Summing this over all suffixes gives the total number of distinct substrings.

Let a[1, n] is an array with n >= 1
We want to calculate the sum of minimums of all subarrays of a, i.e, 

S = summation over 1 <= i <= j <= n of min(a[r] for i <= r <= j)

The method to do this is the contribution technique with monotonic stacks. Instead of iterating over all subarrays, compute:

	For how many subarrays is a[i] the minimum? Let this number be c[i], for 1 <= i <= n

Then, S = summation (1 <= i <= n) c[i] * a[i]

For each 1 <= i <= n, define:

L[i] = the largest index j < i such that a[j] < a[i]
     = 0 if no such index exists

R[i] = the smallest index j > i such that a[j] <= a[i]
     = n + 1 if no such index exists

We count the subarrays where a[i] is the chosen minimum.

A valid subarray must:
- start in [L[i] + 1, i]
  since extending further left would include an element smaller than a[i]

- end in [i, R[i] - 1]
  since extending further right would include an element <= a[i]

Thus, the number of subarrays where a[i] contributes as the minimum is

c[i] = (i - L[i]) * (R[i] - i)

So the final answer is

Σ a[i] * c[i]

Why does this count every subarray exactly once?

Take any subarray a[l, r], and let b be its minimum value.
Let k be the rightmost position in [l, r] such that a[k] = b.

Then:
- there is no element < a[k] inside the subarray
- there is no element <= a[k] to the right of k inside the subarray

Hence the subarray satisfies the conditions for k and is counted in c[k].

Now consider any earlier position x < k with a[x] = b.
Since there is an equal value to the right of x, we have R[x] <= k, so the subarray cannot extend far enough right to include k.
Therefore the same subarray is not counted in c[x].

Thus every subarray is counted exactly once.

11.

Let s[1, n] is a string for n >= 1
Let sa[1, n] is the suffix array of s
Let rank[1, n] is the rank array of s
Let lcp[1, n] is the lcp array of s

We have to calculate

S = summation (1 <= i, j <= n and i != j) LCP(i, j)

Since for any 1 <= i, j <= n, we have LCP(i, j) = LCP(j, i)
S = 2 * S' where

S' = summation (1 <= i < j <= n) LCP(i, j)

Now, LCP(i, j) = min(lcp[r] for ri + 1 <= r <= rj) where ri = min(rank[i], rank[j]) and rj = max(rank[i], rank[j]) for 1 <= i, j <= n
Also, rank[i] = rank[j] iff i = j for 1 <= i, j <= n
Also, if 1 <= i <= n, rank[i] takes on each value in the set [1, n] exactly once
Thus, S = summation (1 <= i <= n, 1 <= j <= n, i != j) LCP(i, j)
				= summation(1 <= i <= n, 1 <= j <= n, i != j) min(lcp[r] for ri + 1 <= r <= rj)
				= summation(1 <= rank[i] <= n, 1 <= rank[j] <= n, rank[i] != rank[j]) min(lcp[r] for min(rank[i], rank[j]) + 1 <= r <= max(rank[i], rank[j])) (since i and j are independent summations)

				= summation(1 <= i <= n, 1 <= j <= n, i != j) min(lcp[r] for min(i, j) + 1 <= r <= max(i, j))

Let A(i, j) = min(lcp[r] for min(i, j) + 1 <= r <= max(i, j)) for 1 <= i <= n, 1 <= j <= n and i != j
Then, A(i, j) = A(j, i) for 1 <= i <= n, 1 <= j <= n and i != j
Thus, S = summation (1 <= i <= n, 1 <= j <= n and i != j) A(i, j)
				= 2 * summation (1 <= i < j <= n) A(i, j)

Let S'' = summation (1 <= i < j <= n) A(i, j)
Thus S' = S''

Now, S'' = summation (1 <= i < j <= n) min (lcp[r] for i + 1 <= r <= j)
Let p = i + 1, then,

S'' = summation (2 <= p <= j <= n) min (lcp[r] for p <= r <= j)
	= summation (2 <= i <= j <= n) min (lcp[r] for p <= r <= j)
	= summation (1 <= i <= j <= n) min (lcp[r] for i <= r <= j) since lcp[1] = 0 < lcp[r] for 1 < r <= n

Thus, S'' is the sum of minimums of all subarrays of lcp
Thus, S' = S'' and S = 2 * S' = 2 * S''
